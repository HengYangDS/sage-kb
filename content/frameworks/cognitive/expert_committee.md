## 5. Five-Level Decision Ladder

### 5.1 Level Definitions

#### Level 1 - Micro (15 minutes)
- **Roles**: 2-3 core roles
- **Angles**: 4-5 core angles
- **Scenarios**: Bug fixes, small features, documentation updates
- **Cost**: Low

#### Level 2 - Rapid (30 minutes)
- **Roles**: 3-5 core roles
- **Angles**: 6-8 core angles
- **Scenarios**: Feature development, performance optimization, security hardening
- **Cost**: Medium-Low

#### Level 3 - Major (2 hours)
- **Roles**: 7-10 core + 2-4 extended roles
- **Angles**: 8-10 core + 2-4 extended angles
- **Scenarios**: Architecture changes, tech stack upgrades, refactoring plans
- **Cost**: Medium-High

#### Level 4 - Strategic (Half day)
- **Roles**: 10 core + 5-8 extended + 1-2 domain
- **Angles**: 10 core + 5-8 extended + 1-2 domain
- **Scenarios**: Product direction, tech selection, large-scale transformation
- **Cost**: High

#### Level 5 - Transformative (1 day)
- **Roles**: All relevant roles (15-25 total)
- **Angles**: All relevant angles (15-25 total)
- **Scenarios**: Company strategy, disruptive changes
- **Cost**: Very High

### 5.2 Level Selection Guide

| Factor | Level 1 | Level 2 | Level 3 | Level 4 | Level 5 |
|--------|---------|---------|---------|---------|---------|
| **Code Impact** | <100 lines | <1K lines | <10K lines | <100K lines | System-wide |
| **Team Impact** | 1 person | 2-3 people | Team | Multi-team | Organization |
| **Timeline** | Days | 1-2 weeks | 1-2 months | Quarter | Year+ |
| **Reversibility** | Easy | Moderate | Difficult | Very Difficult | Strategic |
| **Risk Level** | Low | Medium | High | Very High | Critical |

## 6. Dynamic Activation Algorithm

### 6.1 Auto-Classification Formula

```python
def determine_level(scenario):
    impact_score = calculate_impact(scenario.code_lines, scenario.users, scenario.modules)
    complexity_score = calculate_complexity(scenario.tech_stack, scenario.dependencies)
    risk_score = calculate_risk(scenario.reversibility, scenario.business_impact)
    
    total_score = impact_score * 0.3 + complexity_score * 0.3 + risk_score * 0.4
    
    if total_score < 20: return Level.MICRO
    elif total_score < 40: return Level.RAPID
    elif total_score < 60: return Level.MAJOR
    elif total_score < 80: return Level.STRATEGIC
    else: return Level.TRANSFORMATIVE
```

### 6.2 Dynamic Scaling

**Scale Up When**:
1. New risks discovered during deliberation
2. Knowledge gap identified (no expert coverage)
3. Consensus cannot be reached
4. Stakeholder requests additional review

**Scale Down When**:
1. Clear consensus reached early
2. Problem simpler than anticipated
3. Strong precedent exists
4. Time constraints require focused analysis
