## 2. Committee Patterns

| Pattern           | Experts | Use Case           |
|-------------------|---------|--------------------|
| **Quick Review**  | 3       | Simple decisions   |
| **Standard**      | 5-7     | Normal complexity  |
| **Comprehensive** | 9-12    | Major decisions    |
| **Full Board**    | 15+     | Critical/strategic |

## 3. Selection Process

### 3.1 By Problem Type

| Problem Type | Key Experts                      |
|--------------|----------------------------------|
| Architecture | Architect, Security, Performance |
| Feature      | Product, UX, Tech Lead           |
| Bug fix      | QA, Domain Expert, Support       |
| Security     | Security, Compliance, Risk       |
| Performance  | Performance, DBA, DevOps         |

### 3.2 Selection Criteria

1. Relevance to problem domain
2. Complementary perspectives
3. Potential conflict identification
4. Decision authority level

---

## 4. Analysis Process

### 4.1 Individual Analysis

Each expert provides:

- Perspective on the problem
- Key concerns from their domain
- Recommendations
- Risk assessment

### 4.2 Synthesis

| Step | Action                   |
|------|--------------------------|
| 1    | Collect all perspectives |
| 2    | Identify agreements      |
| 3    | Highlight conflicts      |
| 4    | Weigh trade-offs         |
| 5    | Form recommendation      |
