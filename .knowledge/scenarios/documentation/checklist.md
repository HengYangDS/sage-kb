# Documentation Checklist

> Task checklist for technical writing projects

---

## Planning

- [ ] Target audience identified
- [ ] Documentation scope defined
- [ ] Content outline created
- [ ] Style guide selected
- [ ] Review process established

## Structure

- [ ] Clear hierarchy (H1-H4)
- [ ] Table of contents included
- [ ] Consistent naming conventions
- [ ] Cross-references linked
- [ ] Navigation intuitive

## Content Quality

- [ ] Accurate and up-to-date
- [ ] Concise and clear
- [ ] Jargon explained or avoided
- [ ] Examples provided
- [ ] Edge cases covered
- [ ] Common errors addressed

## Code Examples

- [ ] Syntax highlighted
- [ ] Tested and working
- [ ] Comments explain key points
- [ ] Copy-paste friendly
- [ ] Version/environment specified

## Visual Elements

- [ ] Diagrams where helpful
- [ ] Screenshots current
- [ ] Alt text for images
- [ ] Consistent styling
- [ ] Mobile-friendly display

## API Documentation

- [ ] All endpoints documented
- [ ] Request/response examples
- [ ] Error codes explained
- [ ] Authentication described
- [ ] Rate limits documented

## User Guides

- [ ] Getting started guide
- [ ] Step-by-step tutorials
- [ ] Troubleshooting section
- [ ] FAQ included
- [ ] Glossary if needed

## Technical Reference

- [ ] Configuration options listed
- [ ] Environment variables documented
- [ ] CLI commands described
- [ ] Default values specified

## Maintenance

- [ ] Version history maintained
- [ ] Last updated date shown
- [ ] Feedback mechanism
- [ ] Review schedule set
- [ ] Deprecation notices clear

## Accessibility

- [ ] Plain language used
- [ ] Readable font size
- [ ] Sufficient contrast
- [ ] Screen reader compatible
- [ ] Keyboard navigable

## Publishing

- [ ] Spell check passed
- [ ] Links verified
- [ ] Search functionality
- [ ] PDF export (if needed)
- [ ] Localization (if needed)
